# Resale & Maison Commission System

## 1. Resale logic

- **"Resale starten"** → `POST /api/resale/start` with `userId`, `masterpieceId`, `askingPrice`, `saleMethod` (default `marketplace`).
- System creates a **Resale Commission Agreement** (contract type `resale_commission`) and a **resale_listings** row with status `pending_signature`.
- **Commission by role:** Standard 8%, VIP 6%, Royal/Black 5% (see `RESALE_COMMISSION_RATES` in `server.ts`).
- Resale is **not** activated until the owner has **signed** the agreement. On sign (`POST /api/contracts/sign`):
  - Contract status → signed
  - Resale listing status → `signed`, `signed_at` set
  - Masterpiece status → `resell_pending`

## 2. Contract content (Resale Commission Agreement)

Generated by `generateResaleCommissionAgreement()` and stored in `contracts.content`:

- Asset ID, serial, current owner
- Sale method (Auction / Marketplace)
- Commission %
- Payment flow (escrow → authenticity → Maison release → payout)
- Maison release right, authenticity check requirement, liability disclaimer
- Date; signature block for client

## 3. After sale (`POST /api/resale/complete`)

- Escrow released; ownership and masterpiece valuation updated.
- Commission taken from sale price using listing `commission_pct` (or 8% if no listing).
- **Revenue ledger:** `resale_fee` entry for Maison commission.
- **Resale listing:** `final_sale_price`, `commission_amount`, `status = sold`, `completed_at`.
- **Provenance** and **new certificate** for the new owner.
- **Audit:** `resale_audit_log` with action `resale_completed`.

## 4. Admin

- **Reject:** `POST /api/admin/resale/reject` → listing status `rejected`, masterpiece back to `sold`.
- **Adjust:** `POST /api/admin/resale/adjust` → optional `commissionPct`, `minPrice`.
- **Prioritize auction:** `POST /api/admin/resale/prioritize-auction` → `sale_method = auction`.
- **Approve (list for sale):** existing `POST /api/admin/approve-resale` → masterpiece `available`, `current_owner_id` null.

## 5. Tables

- **resale_listings:** masterpiece_id, seller_id, asking_price, min_price, commission_pct, sale_method, contract_id, status, signed_at, final_sale_price, commission_amount, completed_at.
- **resale_audit_log:** resale_listing_id, action, admin_id, details.

## 6. Audit actions

- `resale_started`
- `contract_signed`
- `resale_rejected`
- `resale_adjusted`
- `prioritize_auction`
- `resale_completed`
